Herhaling

GIT kan wijzigingen in tekst bestanden herkennen en opslaan. Op die manier is het makkelijk om bij te houden wat er veranderd is gedurende het project en ook wie die wijzigingen gemaakt heeft.

Commando's

  • git status
  • git add [file]
  • git add -m "[message]"

GIT houdt die informatie bij in de .git map. (git init) Dit is in principe een soort database met alle informatie er in over de inhoud, de wijzigingen en de status van alle bestanden in de "Working Directory".
In de staging area worden alle bestanden die horen bij een wijziging verzameld. (git add [filename])
Waarna de hele wijziging als één set toegevoegd worden aan de local repository. Voorzien van het juiste commentaar. (git commit -m "Jouw commentaar"])
Ook zonder centrale GIT server is het altijd een goed idee om wijzigingen bij te houden met de bekende commando's. Om wijzigingen te delen met de rest van het ontwikkelteam is zo'n centrale server essentieel.

Nieuw project

Om jouw code te synchroniseren met de server moet je eerst een nieuw project aanmaken op de server. Dit resulteert (bij github) in een link die er uitziet als: https://github.com/YourUserName/YourProjectName.git

Deze link wil je koppelen aan jouw GITBash installatie: 

git remote add [name] [link]

Bijvoorbeeld: 

git remote add origin https://github.com/FrederikBonte/2021_0sv1_team1.git

git push
